你现在是一个资深前端游戏工程师 + 游戏 UI 实现工程师 + 技术美术整合负责人。请为我实现一个战国题材网页策略游戏 MVP，游戏名叫《策问山河》。

这个项目除了基础功能外，还必须包含“可落地的美术资源接入方案”和“类似《杀戮尖塔》的界面体验”，但注意：只能参考其信息架构、布局逻辑、卡牌交互感、地图节点感、整体氛围，不能直接复制《杀戮尖塔》的原始 UI、美术、图标、卡牌版式或角色设计。最终要做成一个原创的“战国竹简纸卷风 roguelike 策略界面”。

## 一、项目目标

请实现一个可以本地运行的前端 MVP，要求：
- React
- TypeScript
- Vite
- Zustand
- TailwindCSS
- 本地 JSON / TS 数据驱动
- localStorage 存档
- 不需要后端
- 不需要登录
- 不需要联网
- 可以使用占位 SVG / CSS 绘制的资源来构建第一版美术表现
- 项目结构清晰，方便后续替换正式美术资源

## 二、界面风格要求（重点）

请将界面设计为“战国版 Slay the Spire 风格”，但必须原创。

### 参考的体验特征（可借鉴）
1. 深色 + 纸卷/旧地图感背景
2. 中央主舞台突出“当前事件 / 当前选择”
3. 底部一排“国策卡”像卡牌手牌一样展示
4. 左右两侧分别放“国家状态 / 士人信息”
5. 有一个节点式地图视图，类似路线推进感
6. 信息层次清晰：标题、叙事、数值、标签、效果一眼能看懂
7. 有适度 hover / press / transition 动效，增强卡牌手感

### 必须避免
- 不要直接复刻《杀戮尖塔》的具体 UI 图、角色图、怪物图、图标、按钮样式
- 不要使用相同的卡牌边框和完全一致的布局细节
- 不要出现任何“看起来就是原游戏资源”的元素

### 我们自己的美术主题
请做成：
- 战国竹简、帛书、地图、铜器、印章、战报、墨色纹理
- 配色以：深褐、暗金、竹青、黯红、灰黑、羊皮纸米色为主
- 质感：古朴、压迫、谋略感、历史感
- 视觉关键词：
  - “国策写在竹简上”
  - “士人像朝堂中的人物条目”
  - “地图像战国形势图”
  - “结算像战报 / 史书记录”

## 四、整体 UI 布局要求

### 1. StartPage
界面像一张展开的古地图 / 战国图卷。
内容包括：
- 游戏标题：《策问山河》
- 副标语：一策可兴邦，一问定山河
- 三个国家选择卡：秦 / 赵 / 齐
- 开始游戏按钮
- 次级按钮：继续游戏 / 重置存档

布局要求：
- 中央偏上是标题
- 中央是国家选择卡
- 底部是按钮
- 有暗纹背景和轻微漂浮粒子或墨迹感（若不做动画，可用背景纹理）

### 2. GamePage
整体布局要接近“策略卡牌主界面”。

推荐布局：
- 顶部：年份、回合、国家名、全局状态提示
- 左侧：国家面板（资源 + 状态）
- 中央：当前事件叙事区 / 回合主舞台
- 右侧：士人面板 / 当前效果 / 状态标签
- 底部：3 张国策卡作为“手牌区”
- 下方或右下角：日志按钮 / 展开战报
- 页面上半区可切换显示简单地图

### 3. CardHand
国策卡必须有“杀戮尖塔式”的可点击手牌感：
- 有卡面标题
- 有类型标签
- 有费用 / 条件 / 效果 / 副作用
- hover 时略微上浮、放大、投影增强
- 点击后有确认反馈
- 不满足条件的卡置灰并说明原因

### 4. AdvisorPanel
士人像“遗物 / 角色特性 + 顾问列表”的结合体：
- 显示头像占位
- 姓名
- 标签（法家/兵家/纵横家/儒家）
- 立即效果摘要
- 长期效果摘要
- 风险提示

### 5. MapView
节点式推进地图，风格类似卡牌 roguelike 地图，但主题是战国形势图。
节点至少包括：
- 咸阳
- 邯郸
- 临淄
- 大梁
- 新郑
- 郢
- 上党
- 长平

地图表现要求：
- 节点之间有连线
- 当前关键节点高亮
- 特殊节点（长平）有战役边框
- 鼠标 hover 能显示节点信息

### 6. BattleView
长平之战界面要更像“战役章节”：
- 顶部：阶段标题
- 中央：战报叙事
- 左右：秦赵态势摘要
- 底部：关键选择按钮
- 有一种“战役卷轴 / 战报书简”的视觉感

### 7. EndingPage
像史书结算：
- 显示结局标题
- 显示结局说明
- 显示关键数值总结
- 显示本局主要士人和关键决策
- 风格像“太史公战国列传的一页摘要”
